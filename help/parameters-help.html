üìä Complete Parameters Reference

Parameter Overview

This comprehensive reference covers every parameter available for MIDI and audio mapping. Each parameter includes its range, purpose, and practical usage tips for live performance and composition.

Parameter Format:
  • parameterName - The exact name used in MIDI mappings
  • [min - max] - The value range
  • Description - What the parameter controls and how it affects visuals

Parameter Categories

  • Grid Parameters: Control the basic structure and layout
  • Animation Parameters: Control movement, rotation, and timing
  • 3D Center Scaling: Create depth effects and 3D illusions
  • Morphing Parameters: Control shape transformations
  • Material Parameters: Control sphere materials and physics
  • Color Parameters: Control overall color and appearance
  • Lighting Parameters: Control scene lighting
  • Post-Processing: Control visual effects and filters
  • Layer Parameters: Control multi-layer system (P5, Shader, etc.)

üî≤ Grid Parameters

These parameters control the fundamental structure of the visual grid - the foundation upon which all animations and effects are built.

Parameter Range Description Performance Tips
gridWidth [1 - 30] Number of shapes horizontally across the grid Higher values create denser patterns but may impact performance
gridHeight [1 - 30] Number of shapes vertically down the grid Balance with gridWidth for optimal performance
cellSize [0.5 - 2.0] Size of individual grid cells/shapes Great for rhythmic scaling effects when mapped to audio
compositionWidth [1 - 50] Overall width of the composition viewport Adjust for different screen sizes and aspect ratios
compositionHeight [1 - 50] Overall height of the composition viewport Works well with compositionWidth for framing effects
randomness [0 - 1] Amount of random variation in grid positioning Low values for structured patterns, high for organic chaos
Grid Performance: Total shapes = gridWidth √ó gridHeight. Keep under 400 total shapes (20√ó20) for smooth 60fps performance on most systems.

üé¨ Animation Parameters

Animation parameters control how shapes move, rotate, scale, and change over time. All animations sync to musical timing based on the global BPM.

Global Animation Controls

Parameter Range Description Performance Tips
globalBPM [60 - 300] Master tempo for all animations (beats per minute) Sync to your music's BPM or use MIDI clock for automatic sync
animationType [0 - 127] Selects different animation styles and patterns Higher values select different animation algorithms

Movement Animation

Parameter Range Description Musical Timing
movementAmplitude [0.01 - 0.5] How far shapes move from their center position Amplitude scales with musical timing
movementDivision Musical Divisions How often movement patterns change (32nd note to 8 bars) Quarter note = 1 beat, Half note = 2 beats, etc.

Rotation Animation

Parameter Range Description Musical Timing
rotationAmplitude [0.01 - 2.0] How much shapes rotate around their center Full rotation = 2œÄ radians ‚âà 6.28
rotationDivision Musical Divisions How often rotation patterns change Faster divisions create spinning effects

Scale Animation

Parameter Range Description Musical Timing
scaleAmplitude [0.01 - 2.0] How much shapes grow and shrink 1.0 = normal size, 2.0 = double size
scaleDivision Musical Divisions How often scaling patterns change Sync to kick drums with quarter notes

Shape Cycling

Parameter Range Description Musical Timing
shapeCyclingDivision Musical Divisions How often shapes change type automatically Slower divisions (bars) work well for gradual changes
shapeCyclingPattern [0 - 10] Pattern for how shapes cycle through types Different algorithms for shape selection
shapeCyclingDirection [0 - 3] Direction shapes cycle (forward, reverse, random) 0=forward, 1=reverse, 2=random, 3=ping-pong
shapeCyclingSync [0 - 1] Whether all shapes change together or individually 0=individual timing, 1=synchronized changes
shapeCyclingIntensity [0 - 2] Strength of the cycling effect Higher values create more dramatic changes
shapeCyclingTrigger [0 - 1] Manual trigger for shape cycling Use with MIDI notes for manual control
Musical Divisions Available:
  • 32nd Note: Very fast (1/8 beat) - 0.0625s at 120 BPM
  • 16th Note: Fast (1/4 beat) - 0.125s at 120 BPM
  • 8th Note: Medium-fast (1/2 beat) - 0.25s at 120 BPM
  • Quarter Note: Medium (1 beat) - 0.5s at 120 BPM
  • Half Note: Slow (2 beats) - 1s at 120 BPM
  • Whole Note: Very slow (4 beats) - 2s at 120 BPM
  • 1-8 Bars: Very long phrases for gradual changes

üéØ 3D Center Scaling Parameters

Center scaling creates 3D depth effects by scaling shapes based on their distance from the grid center, creating dome or bowl-like visual effects.

Parameter Range Description Visual Effect
centerScalingEnabled [0 - 1] Toggles the 3D center scaling effect 0=off, 1=on - use with MIDI notes for toggle
centerScalingIntensity [0 - 2] Strength of the scaling effect Higher values create more dramatic 3D depth
centerScalingCurve [0 - 3] Mathematical curve for scaling calculation 0=linear, 1=exponential, 2=logarithmic, 3=sine wave
centerScalingRadius [0.1 - 5.0] Maximum distance where scaling applies Larger values affect more of the grid
centerScalingDirection [0 - 1] Direction of scaling effect 0=convex (dome), 1=concave (bowl)
centerScalingAnimation [0 - 1] Enables animated scaling effects Creates pulsing or breathing effects
centerScalingDivision Musical Divisions Timing for center scaling animations Sync scaling pulses to musical beats
centerScalingAnimationSpeed [0.1 - 5.0] Speed multiplier for scaling animations Higher values create faster pulsing
centerScalingAnimationType [0 - 3] Type of scaling animation pattern Different algorithms for animated scaling
3D Effect Tips:
  • Convex (dome): Center shapes larger, edges smaller - creates rising effect
  • Concave (bowl): Center shapes smaller, edges larger - creates depth effect
  • Animated scaling: Great for bass-driven pulsing effects
  • Curve types: Exponential for sharp transitions, logarithmic for gradual, sine for smooth waves

üíé Material Parameters

Material parameters control the physical properties of refractive spheres, creating realistic glass, crystal, and liquid effects.

Parameter Range Description Visual Effect
sphereRefraction [0 - 2] Light bending through sphere material 1.0=glass, 1.33=water, 2.0=diamond-like
sphereTransparency [0 - 1] How see-through the spheres are 0=opaque, 1=fully transparent
sphereTransmission [0 - 1] Light transmission through the material Higher values allow more light through
sphereRoughness [0 - 1] Surface roughness of sphere material 0=mirror smooth, 1=completely rough/matte
sphereMetalness [0 - 1] How metallic the sphere surface appears 0=dielectric (glass), 1=metallic (chrome)
sphereScale [0.1 - 3.0] Overall size of refractive spheres Larger spheres have more dramatic refraction
sphereClearcoat [0 - 1] Clear protective coating over base material Creates car paint or lacquered wood effect
sphereClearcoatRoughness [0 - 1] Roughness of the clearcoat layer Independent roughness control for layered materials
sphereEnvMapIntensity [0 - 3] Strength of environment map reflections Higher values create stronger reflections
sphereDistortionStrength [0 - 1] Amount of distortion effect in sphere refraction Creates water ripple or heat haze effects
sphereHighPerformanceMode [0 - 1] Toggles performance optimization for spheres 0=high quality, 1=optimized for performance
Material Combinations:
  • Glass: Refraction=1.5, Transparency=0.9, Transmission=1.0, Roughness=0.0
  • Water: Refraction=1.33, Transparency=0.8, Transmission=0.9, Roughness=0.1
  • Crystal: Refraction=2.0, Transparency=0.95, Transmission=0.8, Roughness=0.0
  • Chrome: Metalness=1.0, Roughness=0.1, Transparency=0.0

üé® Color Parameters

Color parameters control the overall color grading and appearance of the entire composition.

Parameter Range Description Performance Tips
colorHue [0 - 360] Overall hue shift for the entire scene 0=red, 120=green, 240=blue, 360=red again
colorSaturation [0 - 2] Color intensity and vividness 0=grayscale, 1=normal, 2=oversaturated
colorBrightness [0 - 2] Overall brightness/exposure of the scene Great for creating fade-in/out effects
colorContrast [0 - 2] Difference between light and dark areas Higher values create more dramatic lighting
shapeColor [0 - 360] Base hue for shape colors Combined with global color grading
backgroundColor [0 - 360] Background color hue Creates color contrast with shapes
gridColor [0 - 360] Color of grid lines (when visible) Subtle accent color for grid structure
Color Theory Tips:
  • Complementary colors: Hues 180¬∞ apart (red/cyan, blue/orange)
  • Analogous colors: Hues close together (red/orange/yellow)
  • Monochromatic: Same hue with different saturation/brightness
  • Audio mapping: Map hue to frequency ranges for color-reactive visuals

üí° Lighting Parameters

Lighting parameters control the 3D scene lighting, affecting how shapes appear and cast shadows.

Parameter Range Description Visual Effect
ambientLightIntensity [0 - 2] Overall ambient lighting level Base illumination that affects all surfaces equally
directionalLightIntensity [0 - 3] Main directional light (like sunlight) Creates strong shadows and highlights
pointLight1Intensity [0 - 3] First point light source intensity Localized lighting from specific position
pointLight2Intensity [0 - 3] Second point light source intensity Additional accent lighting
rimLightIntensity [0 - 3] Edge lighting that outlines shapes Creates dramatic silhouette effects
accentLightIntensity [0 - 3] Additional accent lighting Highlights specific areas or creates mood
lightColour [0 - 360] Overall color temperature of lighting Warm (red) to cool (blue) lighting moods
Lighting Setups:
  • High Key: High ambient + low directional = soft, even lighting
  • Low Key: Low ambient + high directional = dramatic shadows
  • Rim Lighting: High rim + low ambient = silhouette effects
  • Reactive Lighting: Map light intensities to audio for pulsing effects

‚ú® Post-Processing Effects

Post-processing effects are applied after the 3D scene is rendered, adding cinematic and artistic effects.

Bloom Effect

Parameter Range Description Performance Impact
bloomStrength [0 - 3] Intensity of the glow effect Higher values = more GPU usage
bloomRadius [0 - 1] Size/spread of the bloom glow Larger radius = more GPU usage
bloomThreshold [0 - 1] Brightness level where bloom begins Lower threshold = more pixels affected

Chromatic Aberration

Parameter Range Description Visual Style
chromaticIntensity [0 - 0.1] Amount of color separation effect Creates vintage lens or VHS-style distortion

Vignette Effect

Parameter Range Description Cinematic Use
vignetteIntensity [0 - 1] Darkness of edge vignetting Focuses attention toward center
vignetteRadius [0 - 1] Size of the vignette area Smaller radius = more dramatic effect
vignetteSoftness [0 - 1] Softness of vignette edges Higher values = more gradual transition

Film Grain

Parameter Range Description Aesthetic
grainIntensity [0 - 1] Amount of film grain noise Creates vintage film or analog video look
Performance Tips:
  • Bloom: Most expensive effect - reduce strength/radius for better performance
  • Chromatic Aberration: Minimal performance impact
  • Vignette: Very lightweight effect
  • Film Grain: Lightweight but can cause visual noise at high settings

üé≠ Layer Parameters

Layer parameters control the multi-layer system, allowing different visual elements to be composited together.

P5 Layer Parameters

P5 layer parameters are dynamically created by your p5.js sketches using the p5Param() function.

Parameter Format Range Description Example
p5:parameterName Custom Range Parameters exposed by your p5.js sketch p5:speed, p5:size, p5:color
P5 Parameter Examples:
  • p5:speed - Animation speed in your sketch
  • p5:size - Size of objects in your sketch
  • p5:color - Color hue for sketch elements
  • p5:particleCount - Number of particles in particle system
